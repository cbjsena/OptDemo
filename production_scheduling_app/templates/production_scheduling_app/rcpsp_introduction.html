프로젝트 일정 및 자원 제약 스케줄링 (RCPSP)

1. 문제 설명

자원 제약 프로젝트 스케줄링 문제(Resource-Constrained Project Scheduling Problem, RCPSP)는 여러 개의 활동(Activity)으로 구성된 프로젝트의 일정을 계획하는 문제입니다. 이 문제는 각 활동의 소요 시간과 활동 간의 **선후 관계(Precedence Constraint)**뿐만 아니라, 프로젝트 수행에 필요한 **제한된 자원(Resource Constraint)**을 동시에 고려한다는 점에서 매우 현실적이고 복잡합니다.

문제의 목표는 주어진 제약 조건들을 모두 만족시키면서 프로젝트의 전체 완료 시간(Makespan)을 최소화하는 최적의 활동 시작 시간을 찾는 것입니다.

2. 산업 응용 분야

RCPSP는 자원과 시간의 제약이 중요한 모든 종류의 프로젝트 관리에 핵심적으로 사용됩니다.

  • 건설 프로젝트: 건물이나 교량을 건설할 때, '기초 공사 → 골조 공사 → 마감 공사'와 같은 선후 관계와 함께, 제한된 수의 인력, 크레인, 트럭 등의 자원을 고려하여 최단 공사 기간을 계획합니다.
  • 신제품 개발: '시장 조사 → 제품 설계 → 시제품 제작 → 테스트' 등 일련의 개발 활동을 수행할 때, 각 단계에 필요한 연구원, 디자이너, 엔지니어 등의 인적 자원을 효율적으로 배분하여 출시 시점을 앞당깁니다.
  • IT 시스템 구축: '요구사항 분석 → 시스템 설계 → 개발 → 테스트 → 배포'의 각 단계에 필요한 개발자, 테스터, 서버 등의 자원을 고려하여 프로젝트 일정을 최적화합니다.
  • 대규모 플랜트 유지보수: 정해진 기간 동안 발전소나 공장의 가동을 멈추고 여러 유지보수 활동을 수행할 때, 제한된 유지보수 인력과 특수 장비를 활용하여 작업 중단 시간을 최소화합니다.

3. 수학적 모델

주요 파라미터:

$$ d_j = \text{The duration of activity } j $$

$$ Pred(j) = \text{The set of immediate predecessors of activity } j $$

$$ r_{jk} = \text{The amount of resource } k \text{ required by activity } j $$

$$ R_k = \text{The total availability of resource } k \text{ at any time} $$

결정 변수:

$$ S_j = \text{The start time of activity } j $$

목표 함수:

$$ \min S_{n+1} $$

$$ \text{(Minimize the start time of the dummy 'end' activity, i.e., the project makespan)} $$

주요 제약 조건:
  • 선후 관계 제약 (Precedence Constraint): 선행 활동($i$)이 완료되어야만 후행 활동($j$)을 시작할 수 있습니다.

    $$ S_i + d_i \le S_j \quad \forall i \in Pred(j) $$

  • 자원 제약 (Resource Constraint): 임의의 시간($t$)에 동시에 진행 중인 모든 활동들이 사용하는 자원의 총합은 가용 자원의 양($R_k$)을 초과할 수 없습니다.

    $$ \sum_{j: S_j \le t < S_j+d_j} r_{jk} \le R_k \quad (\forall t, \forall k) $$

해법 (Solution Methods):

RCPSP는 Job Shop 스케줄링보다 더 일반화된 문제로, 대표적인 NP-hard 문제입니다. 이 문제는 다양한 우선순위 규칙(Priority rule)을 기반으로 한 휴리스틱 방법이나, 유전 알고리즘(Genetic Algorithm) 등의 메타휴리스틱으로 근사해를 찾을 수 있습니다. 최적해를 찾기 위해서는 제약 프로그래밍(Constraint Programming, CP)이 매우 효과적이며, OR-Tools의 CP-SAT 솔버는 RCPSP를 모델링하고 해결하는 데 강력한 기능을 제공합니다.
